Homework app challenge 
Thank you for your interest in joining the Advanced Technologies Center as a full­stack 
developer.  The following is a project that will provide a flavor of the work that you’ll be doing at 
ATC. Use this as an opportunity to show us your creativity, problem solving skills, and 
programming ability. 

The goal​
: To complete this stage of the interview, you will need to develop a web application.  

Your client​
: Anyone at ATC who wishes to explore a particular city, or an arbitrary region of a 
city, with regard to trip volume, common pickups or dropoffs, common routes, high or low 
speeds, or other factors you can think of. The client should be able to select an arbitrary region 
within a given city and see one or more analyses. Assume that your clients are not coders, so 
design and build with that in mind.  

Preferred languages​
: Python, Javascript 
Suggested frameworks​
: Flask, Node, React.js, Angular.js, WebGL, d3.js (If you can create an 
amazing app in another language, feel free to do so but note that we mostly use Python and 
Javascript for web apps.)

Sample Data 
Your sample data comes from New York Uber data collected by the City Taxi and Limousine 
Commission (TLC) and FOIL’ed by fivethirtyeight. It is available on ​
Github​
. Please use the April 
2014­­September 2014 data only. Because the data includes only pickup location, you should 
apply a transformation such that half the trips become dropoff locations for the other half of the 
trips. 

For example, you could take rows 1, 3, 5, and assign their pickup locations as drop­off locations 
for rows 2, 4, 6. In this way you should end up with half of the original data (i.e., once a row’s 
pickup location becomes the drop­off location for another row, don’t continue to use it as a 
pickup instance). 

Your final data set should have these columns: 
● Date/Time 
● Lat 
● Lon 
● DropoffLat (taken from another row’s ‘Lat’) 
● DropoffLon (taken from another row’s ‘Lon’) 
● Base (you can ignore this column)
